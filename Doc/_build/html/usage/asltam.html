ASLTAM¶

Il s’agit du module principal de notre programme, il contient toutes les fonctions nécessaires à l’aboutissement du projet.

Module map¶

Le but principal de ce module est l’affichage d’une carte folium, affichant le trajet entre deux gares, ainsi que des descriptions de la route les séparant comme le prix, ou la distance totale. La fonction principale permettant d’afficher la carte peut s’appeler directement après importation de asltam.

asltam.map.carte_interactive.trajet(DEPART, ARRIVEE, data_geo, data_price, data_dist, KEY)[source]¶

Affiche une carte intéractive avec le trajet entre DEPART et ARRIVEE.

Warning

Les données géographiques doivent être dans le bon format (voir Format des données géographiques).

Tip

Il est conseiller d’avoir sa propre clef API personelle. Pour en obtenir une, vous pouvez aller voir le Guide d’installation de clef API.

Parameters

  • DEPART (str) – Nom de la gare de départ.

  • ARRIVEE (str) – Nom de la gare d’arrivée.

  • data_geo (DataFrame) – Tableau des données géographiques dans le format standard.

  • data_price (DataFrame) – Tableau des prix.

  • data_dist (DataFrame) – Tableau des distances.

  • KEY (str) – Clé API.

Returns

  • m (folium.Map) - Carte intéractive affichant une route entre DEPART et ARRIVEE.

Examples
>>> import asltam as atm
>>> prix = atm.load_price().save_as_price(index=' ')
>>> dist = atm.load_dist().save_as_dist(index=' Nom gare ')
>>> geo = atm.load_geo().save_as_geo(index=' Nom gare ')
>>> # pour avoir le bon format
>>> geo = geo[['Latt', 'Long']]
>>> atm.trajet('MONTPELLIER', 'PERPIGNAN NORD', geo, prix, dist)

Note

Il est possible de cliquer sur les portions de route. Essayez ! On verra bien ce qu’il s’y cache.

Module price_distribution¶

Dans ce module, des outils d’analyse sont mis à disposition pour faire de la visualisation des données. Par exemple, nous pouvons utiliser la fonction kde_gare de ce module, directement callable après importation du package, pour pouvoir faire une “kernel distribution estimation” sur la distribution des prix au kilomètre.

asltam.price_distribution.distribution.kde_gare(all, data_price, data_dist, start, target, bw)[source]¶

Affiche la distribution des prix au kilomètre entre deux gares de péages sur un trajet donné (ou sur tout le réseau).

Warning

Attention, les données de distance et de prix doivent être compatible pour pouvoir avoir des résultats cohérents. Voir la documentation pour la Compatibilité.

Parameters

  • all (bool) – Si True affiche la distribution sur le DataFrame entier.

  • data_price (DataFrame) – Tableaux de données de prix entre les gares de péage.

  • data_dist (DataFrame) – Tableaux de données des distances entre les gares de péage.

  • start (str) – Nom de la première gare.

  • target (str) – Nom de la seconde gare.

  • bw (float) – Voir doc bw_adjust de seaborn.

Returns

None.

Examples
>>> import asltam as atm
>>> prix = load_price().save_as_price(index=' ')
>>> dist = load_dist().save_as_dist(index=' Nom gare ')
>>> atm.kde_gare(True, prix, dist, 1)

asltam.io.pricedist.get_way(data_dist, start, target)[source]¶

Renvoie une liste contenant les péages entre start et target. Notre méthode se base sur le fait que l’algorithme de Kruskal nous fournit un graphe du réseau routier avec seulement des arêtes entre deux gares successives dont on déduit trivialement la liste avec un algorithme du plus court chemin (ici Dijkstra).

Parameters

  • data_dist (DataFrame) – Tableau de données sous forme de matrice de distance entre toutes les gares.

  • start (str) – La gare de départ (doit être une élément de data_dist.columns).

  • target (str) – La gare d’arrivée (doit être une élément de data_dist.columns).

Returns

  • L (list) - Liste des péages situé sur le trajet autoroutier le plus court de start à target.

Module graph¶

Ce dernier module s’appuie sur deux gros algorithmes issus de la théorie des graphes (i.e. Dijkstra et Kruskal) dans le but de produire un algorithme minimisant le coût d’un trajet en s’autorisant k sortie d’autoroute durant ce dernier. Le problème étant combinatoire (NP complet), il est conseillé de l’utiliser dans des conditions viables : soit avec un trajet assez court, soit avec une contrainte k très petite ou très grande.

asltam.graph.graph.kmin_cost_out(data_price, data_dist, start, target, k)[source]¶

Calcule le trajet de frais minimum avec la contrainte d’au maximum k entrées/sorties d’autoroute. Renvoie le prix, et les gares de sortie. Attention les tableaux de prix et de distances doivent être compatibles.

Parameters

  • data_price (DataFrame) – Tableau des prix.

  • data_dist (DataFrame) – Tableau des distances.

  • start (str) – Gare d’entrée.

  • target (str) – Gare de sortie.

  • k (int) – Nombre maximum d’entrées/sorties.

Returns

  • L (list) - La liste des sorties à prendre.

  • opt (float) - Renvoie le coût totaldu trajet.

Examples
>>> import asltam as atm
>>> prix = atm.load_price().save_as_price(index=' ')
>>> dist = atm.load_dist().save_as_dist(index=' Nom gare ')
>>> start = 'MONTPELLIER'
>>> target = 'LEUCATE'
>>> g, opt = atm.kmin_cost_out(prix, dist, start, target, 4)
>>> print(g, opt)
['MONTPELLIER', 'SETE', 'AGDE', 'BEZIERS OUEST', 'LEUCATE'] 9.0
